# EMSimLab 路线图（从 UI 骨架到可用产品）

## 最短闭环（建议 1 天内跑通）
1. 模板树能显示（templates.registry → UI 树）
2. 选择模板能切换参数面板（ui.setPanelsForTemplate）
3. 改一个参数，能立刻回写并渲染占位（ui.buildPayload → params.validate → ui.applyPayload）
4. Play/Reset 按钮可用（control.onPlay/onReset；先不做真实物理）

## 关键约定（保证可读性）
- UI 回调不写物理：回调只转发到 `control.*`
- 只有两种数据结构在层间流动：`params` 与 `state`
- 参数合法性只由 `params.validate` 保证（不要在各处散落 clamp）
